AI 老師課堂：如何與 AI 協作開發「非 AI 型防封鎖混淆器」？

這份教材適合直接複製，作為顧問、講師授課的教學簡報與引導講稿。

各位學員、AI 架構師大家好，我是你們的 AI 協作導師。

這堂課的目標非常純粹：「如何引導 AI 在不消耗大模型 Token（零成本）的前提下，寫出極致安全、具備前端視覺化診斷，且完美適應手機端 RWD 的 LINE 防封鎖工具？」

這不只是一個單純的代碼專案，更是一次檢驗「人機協調、精實架構」的經典實戰案例。

🛠️ 三階段遞進式提示詞工法 (Prompting Strategy)

在與大型語言模型 (LLM) 協作大型或精緻專案時，直接丟出所有需求往往會導致 AI 遺漏細節或程式碼混亂。我們需要採用以下三階引導技巧：

1

第一階段：演算法底層探索與對齊

先確認 AI 是否理解「繞過防封鎖」的安全與隱寫術原理，要求給出底層 Core Function。

展開查看「第一階段提示詞範本」 
【角色設定】你是一位資深的網路安全專家與前端架構師。
【技術挑戰】我需要設計一個「完全不需要呼叫 AI API」的 JavaScript 文本擾亂演算法。
【技術規格要求】請幫我寫出以下幾種非 AI 的核心擾亂函數：
  1. 零寬字元隱碼干擾 (隨機不規律在字與字間插入 \u200B, \u200C)
  2. 同形異碼代換 (將英文 a-z 和 0-9 替換為外觀幾近相同但 Unicode 不同的字元)
  3. 中文字交織輕微標點符號 (如在關鍵字間插入 ·)
  4. 能保護原始文字中的 https:// 連結，絕對不能破壞連結的完整度與可點擊性。
請先給我這套核心 JavaScript 演算法代碼，並使用繁體中文為代碼加上詳細註解。
2

第二階段：使用者介面（UI）與 RWD 視覺診斷建構

將邏輯整合進 RWD 網頁，並新增「Steganography Visualizer（隱碼特徵視覺化）」自證清白。

展開查看「第二階段提示詞範本」 
【接續任務】非常好，演算法邏輯非常清晰。現在，我希望把這套演算法做成一個「單檔案 HTML」的精緻測試沙盒。
【介面風格要求】
  - 使用 Tailwind CSS 打造高階科技感 UI。
  - 必須是 RWD 響應式排版：在手機端 (Mobile-first) 必須是單行垂直佈局，在桌面端 (xl Breakpoint)則自動展開為左側設定、右側輸出的 7:5 雙欄配置。
  - 所有輸入框和按鈕在手機觸控時，必須有足夠的 Touch Target 大小，且絕不允許產生 any 水平滾動條 (Horizontal Scrolling)。
【殺手級功能：Steganography Visualizer】
  - 我需要在介面中加入一個實時編譯診斷區。
  - 用 HTML 將原本「人類肉眼看不到的零寬字元」高亮顯示為紅色標籤「🔴零寬」；將同形字標註為藍色、將贅字墊片標註為綠色。
  - 讓使用者能一眼看出：肉眼看起來一模一樣的兩段文字，在二進位和 Hash 級別上已經被徹底打碎。
3

第三階段：高動態贅字與版權宣告

引導 AI 完成動態多態數據填充（問候語與動態秒級戳記），以完美收官專案。

展開查看「第三階段提示詞範本」 
【細節優化】請在現有的 HTML 專案中，加入「動態前後贅字墊片」模組。
  - 允許使用者在下拉選單中選擇：前方加入隨機問候/名人名言/系統分流碼；後方加入秒級高動態時間戳 + 防封鎖微型代碼（如：[傳送驗證: #F39A 於 14:32:05]）。
  - 請於網頁 mt-auto 的頁尾部分，加入一個磨砂玻璃質感的簡約 Footer，寫上版權宣告：「© 2026/6/8 ＡＩ就是力量」。
  - 確保整體代碼是一個乾淨、無外部 JS 包依賴（除了 Tailwind CDN）、完全可獨立運行的 Single-File HTML。
